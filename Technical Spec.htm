Game Mechanics

Platform and OS:

This will be a web-based game. The code will be in JavaScript and HTML. This is meant to run on any Operating system that can run Netscape Navigator or the Internet Explorer. The game will also be able to run on any OS with similiar web browsers that support JavaScript code. We will support IE 5.0 and greater and Netscape 4.0 and greater. Any machine that can run IE 5.0 or Netscape 4.0 must be able to run Natural Disaster.

Code Objects:

http://www.EastCoastGames.com/ND/index.htm - This will store all of the HTML and JavaScript code.

http://www.EastCoastGames.com/ND/ - This will store all of the image files and index.htm.

Control Loop:

draw() - This will be the most important function. It will draw the main game screen every time there is an update. This means it will draw the tiles and current player information.

Game Object Data:

Tile[Rows,Columns] - This array will hold the data for all tiles including:
Owner - Who owns the tile, if anybody.
Structure - The strcuture on the tile, if any.
Image - The image to be used as the source.

Players[Player Number] - This array will hold the player data including:
Name - The player's name.
Points - The number of points the player has.
Gold - The number of gold units the player has.
Wood - The number of wood units the player has.
Steel - The number of steel units the player has.
Concrete - The number of concrete units the player has.
Glass - The number of glass units the player has.
TechRes - A booleon flag for wether the player has built a residence.
TechInd - A booleon flag for wether the player has built an Industrial structure.
TechCom - A booleon flag for wehter the player has built a commercial structure.
TechGov - A booleon flag for wether the player has built a government structure.

Contracts[] - This array will hold the contracts available for bidding.

ContractsAvail[] - This array will be used to take contracts out of to replenish the Contracts[] array.

Events[] - This will hold all of the possible events.

Data Flow:

All data will be stored in local memory. It will come from user input, operations, and hard-coded values.
